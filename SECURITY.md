# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in clawdboard, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

Instead, email **jim@morgen.so** with:
- A description of the vulnerability
- Steps to reproduce
- Impact assessment (if possible)

You'll receive a response within 48 hours. We'll work with you to understand the issue and coordinate a fix before any public disclosure.

## Scope

This policy covers:
- The clawdboard web application (clawdboard.ai)
- The CLI tool (`npx clawdboard`)
- The API endpoints

## Data Privacy

clawdboard only collects aggregate usage data (date, token counts, cost, model names). Prompts, code, file paths, and project names never leave your machine. See our [Privacy Policy](https://clawdboard.ai/privacy) for details.
